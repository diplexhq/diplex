# Требования DIplex

## Функциональные требования

### F1: Сканирование кода
- Рекурсивный обход директорий
- Пропуск файлов `_test.go`
- Пропуск директорий `mocks/`

### F2: Распознавание провайдеров
- Функции с префиксом `New`
- Возврат указателя `*Type` или значения `T`
- Опциональный `error` как второй возвращаемый тип
- Generic провайдеры support (`func NewRepo[T User | Order]() *Repo[T]`)
- Concrete instantiation из generic провайдеров через constraint matching

### F3: Распознавание интерфейсов
- Парсинг методов интерфейса
- Встраиваемые (embedded) интерфейсы из scanned source files (не stdlib/external)
- Generic интерфейсы с параметрами типов (`Repository[T any]`)

### F4: Распознавание реализаций
- Публичные методы структур
- Сопоставление интерфейсов по сигнатуре (`compareArgs` с сужением констрейнов)
- Поддержка указателей и значений в receiver
- Generic receiver поддержка (`*Repo[Order]`, `*InMemoryStore[T]`)
- Структурное встраивание (struct embedding)

### F5: Генерация кода
- DI фасад генерация (один `.go` файл на каждую `-di` директорию)
- Связывание зависимостей через `sync.OnceValue`
- Slice collection генерация (`[]Handler` → агрегированный список)
- CamelCase санитизация полей для generic провайдеров
- Уникальные псевдонимы пакетов с избеганием коллизий

### F6: Обработка ошибок
- Panic на отсутствие интерфейса или реализации
- Panic-first policy (это утилита генерации кода)

## Нефункциональные требования

### NF1: Производительность
- Параллельное сканирование (channel-based iterator)
- Конкурентный парсинг файлов (`sync.WaitGroup`)

### NF2: Надёжность
- Интеграционные тесты через `internal/tests/` (источник истины)
- Самопроверка: собрать DI для проекта + собрать DI для `internal/tests/`

### NF3: Удобство использования
- CLI флаги (`-scan`, `-out`, `-skip`, `-module`, `-di`, `-v`, `-s`)
- Verbose режим с деталями генерации

## Архитектурные ограничения

### A1: Совместимость
- Go 1.26+ (текущая версия в go.mod)
- **Без внешних зависимостей** (только stdlib)
- GOOS/GOARCH поддержка

### A2: Сгенерированный код
- Должен компилироваться без ошибок
- Следует Go Code Review Comments
- gofmt форматирование
- Совместимость с go vet, staticcheck

### A3: Расширяемость
- Templating для кастомизации (`text/template`)
- DI facade pattern для расширяемости
- Конфигурация через флаги
